# 2026-04-17 백테스트-실거래 수익 복구 실험 3안

## 목적

2026-04-16 사고 분석과 최근 parity/replay 결과를 기준으로 보면, 현재 과제는 "새 전략을 계속 발명"하는 것이 아니라 **좋은 신호의 기대값이 라이브에서 어디서 훼손되는지 분리해서 복구**하는 것이다.

이 문서는 다음 질문에 답하기 위한 실험 3개를 정리한다.

1. 현재 손익 악화의 주원인이 **체결 방식**인가
2. 현재 ORB/FVG 신호 중 손실 거래는 **품질이 낮은 진입**에 집중되는가
3. 현재 승리 거래의 수익이 **청산 규칙 때문에 과소 실현**되고 있는가

핫픽스 완료 후 바로 실행 가능한 범위만 담고, 아직 증거가 부족한 대규모 전략 교체는 이번 3안에서 제외한다.

## 전제

### 공통 전제

- `reconcile`/`cancel-fill race`/`manual_intervention` 관련 P0, P1 핫픽스가 먼저 완료되어야 한다.
- 실험 기간 동안 실전 기준 진입 stage 는 `15m OR` 단일화가 기본이다.
- 실험 기간 동안 시스템 전체 일일 진입 횟수는 `1회`로 제한한다.
- 비교 기준은 현재 정책 `passive_top_bottom_timeout_30s` 이다.

### 비교 구간

- 특이일 재현: `2026-04-14`, `2026-04-15`, `2026-04-16`
- 단기 검증: 최근 `60` 거래일
- 확인 검증: 최근 `120` 거래일

### 공통 성공 기준

한 실험안이 채택되려면 아래 4개를 함께 만족해야 한다.

1. 총 손익 또는 기대값이 baseline 대비 유의미하게 개선될 것
2. max drawdown 이 baseline 대비 악화되지 않을 것
3. 슬리피지/미체결/abort 비율이 운영 가능한 수준일 것
4. 라이브 구현 복잡도가 현재 운영 리스크를 과도하게 높이지 않을 것

즉, **수익 증가만으로 채택하지 않는다**. 운영 안정성과 정합성이 함께 좋아져야 한다.

## Baseline

현재 baseline 은 아래와 같다.

- 진입 신호: `orb_fvg`
- 실전 진입 stage: 현재는 multi-stage 지원, 실험 중에는 `15m` 고정 권장
- 진입 방식: `gap.top/bottom` 패시브 지정가
- timeout: `30초`
- fill 확인: cancel 후 `balance` 재확인

관련 코드:

- `src/strategy/parity/execution_policy.rs`
- `src/strategy/parity/parity_backtest.rs`
- `src/strategy/live_runner.rs`

## 실험안 1 — 하이브리드 체결 정책

### 가설

현재 수익 악화의 가장 큰 원인은 신호 품질보다 **패시브 지정가 중심의 체결 정책**일 가능성이 높다. 좋은 신호를 늦게 잡거나 놓치고 있으며, parity 백테스트의 touch-fill 가정과도 차이가 크다.

### 규칙

신호는 그대로 두고, 진입 실행만 바꾼다.

- 1차 주문: `gap.top/bottom` 패시브 지정가
- 1차 대기: `2~3초`
- 미체결 시:
  - 현재가 drift 가 `max_entry_drift_pct` 이내이면
  - `best ask + 1 tick` 또는 `best bid - 1 tick` 수준의 시장성 지정가로 1회 전환
- 2차 대기: `2초`
- 여전히 미체결이면 취소 후 **재진입 금지**

핵심은 "무조건 추격"이 아니라 **짧게 패시브를 시도한 뒤, 허용된 슬리피지 안에서만 체결을 완성**하는 것이다.

### 기대 효과

- 좋은 신호의 미체결 손실 감소
- 강한 추세일에서 진입 누락 감소
- 백테스트와 라이브 체결 규칙 차이 축소

### 구현 범위

- `src/strategy/parity/execution_policy.rs`
- `src/strategy/parity/types.rs`
- `src/strategy/parity/parity_backtest.rs`
- `src/strategy/live_runner.rs`

### 측정 지표

- fill rate
- timeout 취소 비율
- 평균 슬리피지
- 체결 후 평균 보유 수익
- trade expectancy
- 특이일(`2026-04-16`) counterfactual 개선 폭

### 채택 기준

- baseline 대비 기대값 개선
- 평균 슬리피지는 늘더라도 fill rate 개선으로 순손익이 우위
- abort/timeout 급증이 없을 것

### 메모

이 실험은 현재 구조를 가장 적게 흔들면서도 기대값을 바꿀 가능성이 가장 높다. **우선순위 1위**다.

## 실험안 2 — 진입 품질 veto 필터

### 가설

현재 손실 거래는 전체 신호에 고르게 분포하지 않고, 특정 패턴의 약한 신호에 집중되어 있을 가능성이 높다. 이미 저장 중인 FVG 메타데이터를 활용하면 **나쁜 진입을 사전에 버리는 필터**를 만들 수 있다.

### 규칙

기존 `orb_fvg` 신호를 유지하되, 아래 조건을 만족하지 않으면 진입하지 않는다.

- `15m OR` 신호만 허용
- 당일 첫 진입 후보만 허용
- `signal_time` 이 너무 늦으면 거부
- `or_breakout_pct` 가 너무 약하면 거부
- `b_body_ratio` 가 너무 낮으면 거부
- `gap_size_pct` 가 지나치게 작거나 크면 거부

초기 threshold 는 코드에 바로 고정하지 말고, 최근 `60` 거래일 분포를 보고 후보 범위를 먼저 정한다.

예시 후보 범위:

- `signal_time <= 10:30`
- `or_breakout_pct >= 0.15% ~ 0.30%`
- `b_body_ratio >= 0.40 ~ 0.60`
- `gap_size_pct` 는 하위 과소 gap, 상위 과대 gap 제거

### 기대 효과

- 질 낮은 추격성 진입 제거
- 손실 거래 수 감소
- 같은 체결 정책에서도 승률 또는 profit factor 개선

### 구현 범위

- `src/strategy/parity/signal_engine.rs`
- `src/strategy/live_runner.rs`
- `src/presentation/routes/strategy.rs`
- 필요 시 모니터링 문서/대시보드 보강

### 측정 지표

- veto 전/후 거래 수
- veto 된 거래의 가상 PnL
- 승률
- profit factor
- 평균 손실폭
- 신호당 진입 전환율

### 채택 기준

- 거래 수가 줄더라도 총 기대값이 유지 또는 개선
- 손실 거래 감소폭이 승리 거래 감소폭보다 클 것
- 운영자가 이해 가능한 단순한 규칙일 것

### 메모

이 실험은 "좋은 신호를 더 찾는 것"이 아니라 **나쁜 신호를 덜 건드리는 것**에 초점이 있다. 전략 몰입 상태에서 가장 놓치기 쉬운 개선축이다.

## 실험안 3 — 승리 거래 수익 확장 청산

### 가설

현재 시스템은 진입보다 청산이 승리 거래의 우측 꼬리를 잘라내고 있을 수 있다. 특히 122630 같은 장초 강추세 ETF 는 고정 RR 만으로는 좋은 날의 이익을 충분히 회수하지 못할 가능성이 높다.

### 규칙

진입은 baseline 또는 실험안 1의 체결 정책을 그대로 두고, 청산만 아래처럼 변경한다.

- `1R` 도달 시 `50%` 부분 청산
- 잔여 물량 stop 을 본절로 이동
- 이후 잔여 물량은 아래 둘 중 하나로 추적
  - 직전 `5분봉` 저점/고점 이탈
  - `VWAP` 이탈
- 신규 진입 cutoff 이후에는 보수적 관리
- 장 종료 전 강제 청산 유지

### 기대 효과

- 강한 추세일의 평균 승리폭 증가
- 손익비 고정 규칙보다 수익 꼬리 회수 개선
- 승률을 크게 훼손하지 않고 expectancy 개선 가능

### 구현 범위

- `src/strategy/parity/position_manager.rs`
- `src/strategy/parity/parity_backtest.rs`
- `src/strategy/live_runner.rs`
- 필요 시 상태 패널에 부분청산/트레일링 상태 노출

### 측정 지표

- 평균 승리 거래 크기
- 최대 승리 거래 크기
- 부분청산 후 본절 전환 비율
- 총 기대값
- give-back 비율

### 채택 기준

- 평균 손실폭 증가 없이 평균 승리폭이 유의미하게 늘 것
- 부분청산/트레일링 로직이 라이브에서 복잡한 race 를 만들지 않을 것

### 메모

이 실험은 손실 거래를 줄이는 축이 아니라 **이미 맞춘 방향에서 더 많이 벌어오는 축**이다. 우선순위는 3개 중 가장 낮지만, 장이 좋은 날 수익곡선을 가장 크게 바꿀 수 있다.

## 권장 실행 순서

세 실험을 동시에 올리면 원인 분리가 불가능하다. 아래 순서를 권장한다.

1. **실험안 1**로 체결 정책부터 검증
2. **실험안 2**로 진입 품질 필터 검증
3. **실험안 3**으로 청산 확장 검증

이 순서는 "수익 왜곡이 큰 층위부터" 처리하는 순서이기도 하다.

## 구현 우선순위 제안

### Phase A — parity 실험 전용

- 실험안 1 정책 추가
- 같은 신호 집합으로 baseline 과 비교
- 최근 `60/120` 거래일 결과 저장

### Phase B — veto 필터 분석

- 기존 `entry_signal`/`abort_entry`/`trade` 로그에서 메타데이터 추출
- 손실 거래 공통 특성 확인
- threshold 후보 2~3개만 선정

### Phase C — 청산 정책 비교

- 승리 거래 샘플 중심으로 부분청산/트레일링 비교
- drawdown 악화 여부 동시 확인

## 채택하지 않은 아이디어

### `orb_vwap_pullback` 전체 전략 교체

방향 자체는 타당하지만, 지금 바로 첫 실험으로 쓰기엔 변경 범위가 크다.

- signal 생성
- armed 상태
- 1분 반전 확인
- 시장성 지정가
- 청산 규칙

이 모든 축이 한 번에 바뀌면 어떤 변화가 수익을 만들었는지 분리하기 어렵다. 따라서 첫 3개 실험은 **현 전략의 알파를 보존한 채 execution, filter, exit 세 축을 따로 검증**하는 데 집중한다.

## 최종 판단

현재 시점에서 가장 가능성이 높은 수익 개선축은 아래 순서다.

1. **하이브리드 체결 정책**
2. **진입 품질 veto 필터**
3. **부분청산 + 추적 청산**

즉, "전략을 완전히 바꾸면 수익이 날 것"보다 "현재 전략의 좋은 신호를 현실적으로 체결하고, 약한 진입을 버리고, 승리 거래를 더 길게 가져가는 것"이 더 빠르고 검증 가능한 경로다.
