## 4. 원리 vs 현재 구현 검증

### 4.1 원리 (사용자 정의)

> 15분봉으로 시작하여 장 시작 9시부터 15분의 고가와 저가를 기준으로 한다. 그리고 15분봉에서 5분봉으로 넘어간다. 상승형 캔들이 고가를 몸통돌파하고 FVG가 함께 생성되었다. 캔들이 FVG로 진입했다면 매수 진입을 시작한다. 손절은 FVG가 형성된 캔들의 이전 캔들 저가에 걸어두고 손익비 1:2나 1:3으로 매수 진입을 진행하면 된다.

## 6. 근본 원인 분석 (두 층위)

### 6.1 Level A — 시스템 아키텍처 결함

> **"시스템 상태가 KIS 실제 상태와 괴리되는 것을 허용하는 아키텍처"**

- cancel API 응답 = "미체결 확정"으로 해석 — KIS가 cancel 수신 전 체결할 수 있다는 가능성을 고려하지 않음
- 체결 확인을 REST 폴링(inquire-daily-ccld, 3초 간격)에 의존 — WS 체결통보(`@2992173` 구독 중이지만 실제로 소비 안 함)
- cancel 후 KIS 잔고 재조회 없음
- Source of truth가 분산되어 있고, 두 상태가 갈라지는 시나리오에서 **KIS 실상태를 우선 신뢰하는 동기화 루프 부재**
- 한 번 갈라지면 시스템이 영원히 복구 못 함 → 오늘 97+89주가 SL/TP/시간청산 어느 것도 못 맞고 2시간 반 이상 방치

### 6.2 Level B — 전략 설계 결함 (더 근본)

> **"추세 판단 없이 FVG 패턴만 감지해서 진입하는 것"**

ORB+FVG 원리의 철학:
- OR = "오늘의 방향 결정 구간"
- OR 돌파 = "방향이 확정됨"
- FVG = "돌파 추세 중 기관이 만들어낸 유동성 갭 (재진입 기회)"

즉 **OR 돌파가 "진짜 추세의 시작"이어야** FVG 진입이 의미 있다. 오늘은:
- 09:15 B 캔들이 OR_high 돌파 ✓ (이론상 Long 시작 지점)
- 그러나 09:35부터 가격이 **OR_high 아래로 다시 떨어짐 → 돌파 실패(false breakout)**
- 이 시점에 "오늘 돌파는 실패였구나" 판단하고 **Long 전략 중단해야 함**
- 대신 인버스가 14:20 OR_high 돌파할 때 Long 진입 (하락 추세 확정 후)

현재 구현은 **False breakout 감지 로직이 없어서**, 실패한 돌파의 뒤끝에서 계속 Long FVG를 찾아 진입.

## 7. "15분 지나면 바로 신호를 잡아야 하는데 안 된다"에 대한 검증

### 7.1 이론적 최소 지연

ORB+FVG 원리상 **3캔들 A-B-C 패턴** 필요:
- A = B 직전 캔들 (SL 기준)
- B = OR 돌파 캔들
- C = B 다음 캔들 (FVG zone 형성 확정)

B 캔들이 **몸통(종가)으로 돌파**를 확인하려면 B가 닫혀야 하고, C가 새로 시작해야 하니 **최소 5분 대기 + 1분 tick = 약 6분 불가피**.

### 7.2 오늘 실제

| 시점 | 이벤트 |
|------|--------|
| 09:15:00 | OR 구간 종료 |
| 09:20:00 | 첫 5분봉(B=09:15 봉)이 OR_high 돌파한 상태로 닫힘 |
| 09:20:00~ | C 캔들 시작, low가 A.high 위면 FVG 유효 |
| **09:21:04** | 첫 entry_signal 발생 |

**지연 6분 4초, 이론 최소치와 거의 일치**. 따라서 "신호가 안 잡힌다"는 것은 타이밍 문제가 아님.

### 7.3 실제 체감 원인 — 약한 신호 과잉

| 시각 | FVG | B.close | 5m OR 돌파 | **15m OR 돌파** | 원리 유효성 |
|------|-----|---------|------------|----------------|-----------|
| 09:20 | #1 | 101,430 | ✓ | ✓ | 정상 |
| 10:10 | #2 | 101,415 | ✓ | ✓ | 정상 |
| 11:55 | #3 | ~101,000 | ✓ | 경계 | 약함 |
| 12:50 | #4 | ~100,700 | ✓ | **✗** | **5m 기준만 통과** |
| 13:20 | #5 | ~100,900 | ✓ | **✗** | **5m 기준만 통과** |
| 14:00 | #6 | ~100,700 | ✓ | **✗** | **5m 기준만 통과** |
| 14:15 | #7 | ~101,000 | ✓ | 경계 | 약함 |

**오후 신호 4건(#4~#7)은 원리적 의미의 15m OR 돌파를 못 이뤘는데, 현재 Multi-Stage 구현이 "5m stage OR"을 허용해서 돌파로 인식한 것**. 원리적으로는 이 신호들 모두 **무효**.

## 9. 수정 방향 (우선순위)

### 9.1 P0 — 즉시 수정 (race condition)

1. **cancel 후 KIS 잔고 재조회 의무화**: `cancel API 응답=성공` ≠ `미체결 확정`. cancel 직후 `/uapi/domestic-stock/v1/trading/inquire-balance` 호출로 실제 보유 수량 검증 후 분기
2. **WS 체결통보(`@2992173`) 우선 신뢰**: 현재 구독 중이지만 소비 안 함 → 체결통보 도착 즉시 cancel 시도 중단, 체결 처리
3. **cancel 함수 매수/매도/TP 분리**: 에러 메시지 "TP 취소 3회 실패"가 매수 cancel에서 나오는 라벨링 오류 (P1 #4)
4. **active_positions 복구 시 KIS 잔고 우선**: 재시작 시 DB가 비어있어도 KIS REST로 보유 종목 발견 시 자동 등록

### 9.2 P1 — 전략 로직 개선

1. **False breakout 필터**: B 캔들 돌파 후 **다음 N개 5분봉이 OR_high 위에서 유지**되어야 진입 자격 부여. B 직후 되돌림으로 OR 안으로 복귀하면 해당 돌파 무효 처리
2. **추세 정합성 체크**: VWAP 또는 이동평균 기울기로 현재 추세 판단, 반대 방향 신호 차단
3. **첫 진입 실패 시 종일 락**: #1이 SL 터치하면 Long 종일 금지 + 인버스 모니터링만
4. **OR 재평가**: 시가가 OR_high 위이거나 OR 내 박스권이면 **당일 진입 아예 포기**

### 9.3 P1 — Multi-Stage 정리

1. **진입 조건을 15m stage OR_high로 고정** — 5m stage는 참고용(UI 표시)으로만 유지
2. 또는 **Multi-Stage 로직 자체 제거** — 원리에 없는 확장이므로 단순화

### 9.4 P2 — 기타
