# 거래량 지표 (Volume Indicators)

## 거래량 분석의 중요성

> "가격은 거짓말을 할 수 있지만, 거래량은 거짓말하지 않는다."

거래량은 가격 움직임의 **신뢰도**를 검증한다. 가격이 상승할 때 거래량도 함께 증가하면 상승세가 진짜이고, 거래량이 감소하면 상승세가 약해지고 있다는 뜻이다.

| 가격 | 거래량 | 해석 |
|------|--------|------|
| 상승 | 증가 | **진짜 상승** — 매수세 유입 확인 |
| 상승 | 감소 | 상승 약화 — 관심 하락, 반전 가능 |
| 하락 | 증가 | **진짜 하락** — 매도세 강함 |
| 하락 | 감소 | 하락 약화 — 매도 소진, 반등 가능 |

## OBV (On Balance Volume, 온밸런스볼륨)

### 의미

Joseph Granville이 개발한 OBV는 거래량의 누적 흐름으로 **매수/매도 압력의 방향**을 측정한다. 가격보다 거래량이 먼저 움직인다는 전제에 기반한다.

### 계산식

매우 단순하다:

```
종가 상승 시: OBV = 전일 OBV + 금일 거래량
종가 하락 시: OBV = 전일 OBV - 금일 거래량
종가 동일 시: OBV = 전일 OBV (변동 없음)
```

### 계산 예시

| 일자 | 종가 | 거래량 | 변동 | OBV |
|------|------|--------|------|-----|
| 1일 | 100 | 50,000 | - | 50,000 (초기값) |
| 2일 | 102 | 60,000 | 상승 | 50,000 + 60,000 = **110,000** |
| 3일 | 101 | 40,000 | 하락 | 110,000 - 40,000 = **70,000** |
| 4일 | 104 | 80,000 | 상승 | 70,000 + 80,000 = **150,000** |
| 5일 | 104 | 30,000 | 동일 | **150,000** |
| 6일 | 103 | 55,000 | 하락 | 150,000 - 55,000 = **95,000** |

### 해석

OBV의 **절대값**보다 **추세 방향**이 중요하다:

| 상황 | 해석 |
|------|------|
| OBV 상승 + 가격 상승 | 상승 추세 **확인** (건강한 상승) |
| OBV 상승 + 가격 하락 | **강세 다이버전스** — 반등 가능성 |
| OBV 하락 + 가격 상승 | **약세 다이버전스** — 상승 후 하락 가능 |
| OBV 하락 + 가격 하락 | 하락 추세 확인 |
| OBV 횡보 | 방향성 없음, 관망 |

### 활용법

- **OBV에 이동평균선 적용**: OBV의 MA를 구해 OBV의 추세 판단
- **OBV 추세선**: 가격 추세선처럼 OBV에도 추세선을 그려 이탈 감지
- **가격 돌파 확인**: 가격이 저항선을 돌파할 때 OBV도 함께 상승하면 진짜 돌파

### Rust 구현

```rust
pub struct Obv {
    current: i64,
    prev_close: Option<f64>,
}

impl Obv {
    pub fn new() -> Self {
        Self {
            current: 0,
            prev_close: None,
        }
    }

    pub fn next(&mut self, close: f64, volume: u64) -> i64 {
        if let Some(prev) = self.prev_close {
            if close > prev {
                self.current += volume as i64;
            } else if close < prev {
                self.current -= volume as i64;
            }
            // 동일하면 변동 없음
        } else {
            self.current = volume as i64; // 초기값
        }

        self.prev_close = Some(close);
        self.current
    }
}
```

- **필요 최소 데이터**: 1개 (즉시 계산 가능)
- **시간복잡도**: O(1) per update
- **공간복잡도**: O(1) (누적값 + 전일 종가만 보관)

## VWAP (Volume Weighted Average Price, 거래량 가중평균가격)

### 의미

VWAP는 거래량으로 가중한 평균 가격이다. **기관 투자자**들이 매매 효율성을 평가하는 벤치마크로 사용한다. "내 체결가가 VWAP보다 좋은가?"를 판단하는 기준.

### 계산식

```
대표가격 (Typical Price) = (고가 + 저가 + 종가) / 3

VWAP = Σ(대표가격 × 거래량) / Σ(거래량)
     = 누적(대표가격 × 거래량) / 누적(거래량)
```

### 계산 예시

| 일자 | 고가 | 저가 | 종가 | 대표가 | 거래량 | 대표가×거래량 | 누적대표가×거래량 | 누적거래량 | VWAP |
|------|------|------|------|--------|--------|-------------|-----------------|-----------|------|
| 1일 | 102 | 98 | 100 | 100.0 | 50,000 | 5,000,000 | 5,000,000 | 50,000 | **100.0** |
| 2일 | 105 | 101 | 103 | 103.0 | 70,000 | 7,210,000 | 12,210,000 | 120,000 | **101.8** |
| 3일 | 104 | 100 | 102 | 102.0 | 40,000 | 4,080,000 | 16,290,000 | 160,000 | **101.8** |

### 해석

| 상황 | 의미 |
|------|------|
| 현재가 > VWAP | **매수세 우위** — 대부분의 거래가 현재가보다 낮은 가격에서 이루어짐 |
| 현재가 < VWAP | **매도세 우위** — 대부분의 거래가 현재가보다 높은 가격에서 이루어짐 |
| 현재가 ≈ VWAP | 균형 상태 |

### 활용법

- **지지/저항선**: VWAP는 장중 동적 지지/저항선으로 작용
- **매매 효율성**: 매수는 VWAP 아래에서, 매도는 VWAP 위에서 하면 시장 평균보다 유리
- **추세 확인**: 가격이 VWAP 위에서 지속적으로 머물면 상승 추세

### 일중 VWAP vs 다일 VWAP

| 종류 | 리셋 주기 | 용도 |
|------|----------|------|
| **일중 VWAP** | 매일 장 시작 시 리셋 | 데이트레이딩, 장중 매매 |
| **다일 VWAP** | 리셋하지 않음 | 중기 추세 분석, 스윙 트레이딩 |

> 일반적으로 VWAP는 **일중** 기준으로 사용한다. 장이 열리면 리셋.

### Rust 구현

```rust
pub struct Vwap {
    cumulative_tp_volume: f64,  // Σ(대표가격 × 거래량)
    cumulative_volume: f64,     // Σ(거래량)
}

impl Vwap {
    pub fn new() -> Self {
        Self {
            cumulative_tp_volume: 0.0,
            cumulative_volume: 0.0,
        }
    }

    /// 장 시작 시 리셋
    pub fn reset(&mut self) {
        self.cumulative_tp_volume = 0.0;
        self.cumulative_volume = 0.0;
    }

    pub fn next(&mut self, high: f64, low: f64, close: f64, volume: u64) -> f64 {
        let typical_price = (high + low + close) / 3.0;
        let vol = volume as f64;

        self.cumulative_tp_volume += typical_price * vol;
        self.cumulative_volume += vol;

        if self.cumulative_volume > 0.0 {
            self.cumulative_tp_volume / self.cumulative_volume
        } else {
            0.0
        }
    }
}
```

- **필요 최소 데이터**: 1개 (즉시 계산 가능)
- **시간복잡도**: O(1) per update
- **공간복잡도**: O(1) (누적합 2개만 보관)
- **입력**: 고가, 저가, 종가, 거래량

## 거래량 지표 활용 원칙

1. **단독 사용 금지**: 거래량 지표는 항상 가격 지표와 함께 사용
2. **확인 도구**: 가격 지표의 시그널을 거래량으로 "확인"하는 용도
3. **돌파 검증**: 가격이 지지/저항선을 돌파할 때 거래량 동반 여부 확인
4. **다이버전스**: 가격과 OBV/VWAP의 괴리는 추세 전환의 선행 신호
